# v1.2.0 Release Notes Draft

Reusable iOS fastlane release template for projects that already manage signing in Xcode.

## Highlights

- Installer now creates `.env` automatically when it is missing
- Multi-project and multi-workspace detection now fails safe instead of leaving misleading defaults
- Installer now prints a recommended explicit command when detection is ambiguous
- Homebrew formula updated to `1.2.0`
- Added tap publishing notes for moving from direct formula installation to a dedicated tap repository
- CI now validates generated `.env` values and multi-project detection behavior

## Installation

Install with `curl`:

```bash
curl -fsSL https://raw.githubusercontent.com/IOSTimor/ios-fastlane-template/main/scripts/install.sh | bash -s -- /path/to/your/ios-project
```

Install with Homebrew:

```bash
brew install --formula https://raw.githubusercontent.com/IOSTimor/ios-fastlane-template/main/Formula/ios-fastlane-template.rb
ios-fastlane-template /path/to/your/ios-project
```

## Complex Project Note

If your repository contains multiple `.xcodeproj` or `.xcworkspace` files, the installer now leaves `PROJECT`, `WORKSPACE`, `SCHEME`, and `APP_NAME` empty until you specify them. It also prints a recommended explicit command.

Example:

```bash
ios-fastlane-template /path/to/your/ios-project \
  --project AppOne.xcodeproj \
  --scheme AppOne \
  --app-name AppOne
```
